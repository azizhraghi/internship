Communication

How Agents Talk

Agents never call each other directly. All communication flows through the Redis Streams event bus following a publish-subscribe pattern.

📡

Event-Driven Messaging

Agents emit events to named streams. Other agents subscribe to streams they care about. The Orchestrateur monitors all streams.

Veille → article.discovered → Bibliométrie
When Veille finds a new article by a lab researcher, Bibliométrie automatically updates their publication list.
🔄

Request-Reply (via Events)

When an agent needs a result from another, it emits a request event and listens for the reply event with a correlation ID.

Optimisation → simulation.request → Simulation
Simulation → simulation.result → Optimisation
Optimisation requests a simulation run and waits for results before computing recommendations.
📢

Broadcast (Fan-out)

Some events are consumed by multiple agents simultaneously. The event bus delivers to all subscribers.

MIS → project.updated → Qualité + Orchestrateur + Bibliométrie
When a project is updated, Qualité checks consistency, Orchestrateur logs the change, and Bibliométrie may update related publications.
👤

Human-in-the-Loop

Sensitive actions go to a review queue. The agent proposes, a human approves/rejects, then the agent executes or aborts.

Any Agent → action.proposed → Review Queue → action.approved → Agent
Publishing a policy brief, deleting data, or modifying budgets always requires human approval.

Error Handling & Recovery

What happens when things go wrong — because they will.

🔄

Automatic Retry

Failed event processing retries with exponential backoff (1s → 2s → 4s → 8s). Max 5 attempts before dead letter.

📬

Dead Letter Queue

Events that exhaust retries go to a DLQ. The Orchestrateur monitors it and escalates to admins.

💓

Heartbeat & Health

Every agent sends heartbeats every 30s. If 3 consecutive heartbeats are missed, Orchestrateur marks the agent as degraded.

🔀

Graceful Degradation

If Veille is down, Bibliométrie still runs from cached data. If Simulation fails mid-run, the scenario state is preserved for restart.

Shared Framework

Common Infrastructure

Every agent inherits from BaseAgent and uses these shared services.

🧬

BaseAgent

Abstract base class providing lifecycle hooks, event handling, health reporting, and audit logging.

class BaseAgent(ABC):
    name: str
    version: str
    permissions: list[str]
    requires_approval: list[str]

    async def start(self): ...
    async def stop(self): ...
    async def handle_event(self, event): ...
    async def emit_event(self, stream, data): ...
    async def propose_action(self, action): ...
    async def log_audit(self, entry): ...
    async def heartbeat(self): ...
📡

Event Bus

Redis Streams wrapper with consumer groups, acknowledgments, and dead-letter handling.

class EventBus:
    async def publish(
        self, stream: str,
        event: Event
    ) -> str:  # returns event ID

    async def subscribe(
        self, streams: list[str],
        group: str,
        handler: Callable
    ): ...

    async def ack(self, stream, group, id): ...
    async def get_pending(self, stream, group): ...
🤖

Mistral AI Client

Unified LLM client with model routing, retry logic, rate limiting, and response caching.

class LLMClient:
    async def complete(
        self,
        prompt: str,
        model: str = "mistral-small",
        temperature: float = 0.3,
        response_format: type = None,

    async def embed(

Schemas

Pydantic v2 base schemas for events, actions, audit entries, and API responses.

class Event(BaseModel):
    id: str = Field(default_factory=uuid4)
    type: str
    source_agent: str
    timestamp: datetime
    correlation_id: str | None
    payload: dict

class AgentAction(BaseModel):
    agent: str
    action_type: str
    requires_approval: bool
    payload: dict
    status: Literal["proposed","approved","rejected","executed"]

Security

JWT token validation, RBAC enforcement, rate limiting, and request signing for inter-agent trust.

# Role-Based Access Control
class Permission(Enum):
    READ_ARTICLES = "veille:read"
    MANAGE_PROJECTS = "mis:manage"
    APPROVE_ACTIONS = "orchestrateur:approve"
    ADMIN = "system:admin"

# Dependency injection guard
def require_permission(perm: Permission):
    async def guard(token=Depends(get_token)):
        if perm not in token.permissions:
            raise HTTPException(403)
